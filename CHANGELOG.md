# Changelog

Все заметные изменения в проекте документируются в этом файле.

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/).

## [1.1.0] - 2025-06-01

### Добавлено
- InquirerPy — стрелочные меню вместо ручного ввода номеров
- `hh-apply init` запоминает прошлые ответы при повторном запуске
- `hh-apply done` — убрать решённую тестовую вакансию из отчёта
- Интерактивный режим `run` — вопросы перед запуском
- Поддержка нескольких профилей конфига (`-c qa.yaml`)

### Улучшено
- Логин через нативный браузер на компе (Chrome, Edge, Brave, Yandex)
- Обработка попапов hh.ru (чат рекрутера, реклама, обязательное письмо)
- Понятные ошибки сети/сессии с подсказками следующего шага
- Обработка database locked с понятным сообщением
- Подробные пошаговые инструкции при логине в браузере
- Валидация зарплаты, regex и подтверждение дефолтного письма в init
- Обработка битого YAML + валидация типов после загрузки
- Dry-run показывает подсказки следующего шага
- Капча возвращает STATUS_SENT если решена

### Исправлено
- Промахи по кнопке отклика
- Чат рекрутера ломал сессию
- Переход на страницу вакансии вместо отклика
- Закрытие рекламных вкладок и попапов
- Login на Windows — системный Chrome вместо Patchright Chromium

## [1.0.0] - 2025-05-01

### Добавлено
- Поиск вакансий по фильтрам (запрос, регион, зарплата, опыт, график)
- Отклики через реальный браузер — isTrusted клики
- Персонализация сопроводительного письма ({company}, {position}, {salary})
- Приоритизация свежих вакансий (сегодня -> вчера -> старые)
- Капча в терминале (Kitty Graphics Protocol, Sixel)
- SQLite база всех откликов с экспортом в CSV/JSON
- Patchright антидетект (CDP patch, fingerprint noise, UA rotation)
- Rich Live-дашборд с прогрессбаром и счётчиками
- Автозапуск по расписанию через crontab
- Поднятие резюме через Android API (boost)
- Мониторинг ответов рекрутеров (responses)
- Команда whoami — информация об аккаунте
- SQL-запросы к базе данных (query)
