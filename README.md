# hh-apply

**Массовые автоматические отклики на вакансии hh.ru из терминала.**

100-150 откликов в день. Без бана. Без паролей. Без подписок.

---

## Зачем это нужно

Доля отказов на hh.ru — 98-99%. ATS отклоняют отклик за 5 секунд, не читая резюме. Единственный способ попасть на собеседование — массовые отклики на все подходящие вакансии. Эта утилита делает это за вас.

## Чем мы лучше аналогов

| | hh-apply | Платные сервисы | Ручные отклики |
|---|---|---|---|
| **Цена** | Бесплатно | 5 000-12 000 руб/мес | Бесплатно |
| **Скорость** | 100-150/день | 5-20/день | 10-20/день |
| **Безопасность** | С вашего устройства, никаких паролей | Сотни аккаунтов с одного сервера | - |
| **Детект** | Patchright + isTrusted клики | Палятся антиботом | - |
| **Данные** | Хранятся локально | На чужом сервере | - |

## Возможности

### Антидетект

| Уровень | Технология | Что решает |
|---------|-----------|-----------|
| CDP | **Patchright** (форк Playwright) | Нет Runtime.enable — невидим для антибот-скриптов |
| Browser | ignore_default_args | Убраны --enable-automation и другие флаги автоматизации |
| Fingerprint | Canvas/Audio/WebGL noise | Уникальный отпечаток при каждом запуске |
| Поведение | isTrusted mouse clicks | Реальные события мыши, не JS-инъекции |
| Задержки | Рандомные паузы 1.5-12 сек | Имитация человеческого ритма |
| API | Android-маскировка | Сервисные команды через официальный API hh.ru |

### Отклики

- Автоматический поиск вакансий по настроенным фильтрам
- Отклик через реальный клик мышью (isTrusted: true)
- Сопроводительное письмо (опционально)
- Обработка модалок, foreign warning, доп. вопросов
- Закрытие рекламных вкладок и попапов
- Проверка сессии каждые 15 откликов
- Graceful shutdown по Ctrl+C с сохранением прогресса

### Фильтры

- **Поисковый запрос** — любой текст
- **Регион** — Москва, СПб, вся Россия, любой город
- **Зарплата** — минимальная, только с указанной
- **Опыт** — без опыта, 1-3, 3-6, 6+ лет
- **График** — удалёнка, офис, смены, гибкий, вахта
- **Занятость** — полная, частичная, проектная
- **Regex исключения** — `--exclude "junior|стажёр|bitrix|хакатон"`
- **Исключения компаний** — по названию
- **Исключения слов** — в названии вакансии

### Капча

Когда hh.ru показывает капчу:
1. Громкий звуковой сигнал
2. Скриншот капчи отрисовывается прямо в терминале (Kitty Graphics Protocol)
3. Вводите текст в терминале — бот вставляет его в браузер
4. Fallback: сохранение картинки в файл, если терминал не поддерживает графику

### Тестовые вакансии

Вакансии с тестовыми заданиями не пропадают — они автоматически сохраняются в файл `~/.hh-apply/test_vacancies.txt` со ссылками. Зайдите и пройдите тесты вручную, правильно.

### Поднятие резюме

```bash
hh-apply boost
```

Одна команда — резюме прыгает наверх в поиске рекрутеров. Через Android API, мгновенно.

### База данных

Все отклики, пропущенные вакансии и причины — в SQLite. Прямой доступ:

```bash
# Тестовые вакансии
hh-apply query "SELECT title, url FROM skipped_vacancies WHERE reason='test_required'"

# Экспорт всех откликов в CSV
hh-apply query "SELECT * FROM applications" --csv -o отклики.csv

# Статистика
hh-apply stats
```

## Безопасность

- **Никаких паролей.** Вы сами логинитесь в браузере — мы сохраняем только куки. Ваши данные не покидают ваше устройство.
- **Запросы с вашего IP.** Не сервис с одного сервера на сотни аккаунтов. Ваш компьютер, ваш IP, один аккаунт.
- **Patchright.** Браузер неотличим от обычного Chrome. Нет CDP-артефактов, нет маркеров автоматизации.
- **Прокси из окружения.** Если нужно — подхватит `HTTPS_PROXY` / `http_proxy` автоматически.

## Быстрый старт

```bash
# 1. Настройка через визард (запрос, фильтры, письмо)
hh-apply init

# 2. Войти в hh.ru (откроется браузер, залогиньтесь сами)
hh-apply login

# 3. Запустить
hh-apply run
```

## Все команды

| Команда | Описание |
|---------|----------|
| `hh-apply init` | Интерактивная настройка |
| `hh-apply login` | Войти в hh.ru через браузер |
| `hh-apply run` | Запустить массовые отклики |
| `hh-apply api-login` | OAuth для API-команд |
| `hh-apply whoami` | Проверка аккаунта |
| `hh-apply boost` | Поднять резюме в поиске |
| `hh-apply stats` | Статистика откликов |
| `hh-apply query` | SQL-запросы к базе |

### Опции `hh-apply run`

| Опция | Описание |
|-------|----------|
| `-c, --config` | Путь к конфигу (по умолчанию: config.yaml) |
| `-l, --limit` | Макс. количество откликов |
| `-e, --exclude` | Regex для исключения вакансий |
| `--headless` | Скрытый режим браузера |
| `--dry-run` | Только поиск, без откликов |
| `-r, --report` | Сохранить отчёт в файл |

## Примеры

```bash
# 150 откликов, исключить стажировки и хакатоны
hh-apply run --limit 150 --exclude "junior|стажёр|intern|хакатон|bitrix"

# Dry run — посмотреть что найдёт
hh-apply run --dry-run

# Поднять резюме
hh-apply boost

# Проверить аккаунт
hh-apply whoami
# 🆔 12345678 Иванов Иван [ 📄 2 | 👁️ +47 | ✉️ +12 ]

# Посмотреть тестовые вакансии для ручного прохождения
cat ~/.hh-apply/test_vacancies.txt

# SQL-запрос
hh-apply query "SELECT count(*) FROM applications WHERE status='sent'"
```

## Конфигурация

`hh-apply init` создаст config.yaml через визард. Можно отредактировать вручную:

```yaml
search:
  query: "python developer"
  area: 1                               # 1=Москва, 2=СПб, 113=Россия
  salary_from: 200000
  salary_only: true
  experience: "between3And6"            # noExperience, between1And3, between3And6, moreThan6
  schedule: ["remote", "fullDay"]       # remote, fullDay, shift, flexible, flyInFlyOut

filters:
  exclude_pattern: "junior|стажёр|bitrix|хакатон"
  exclude_companies: ["Компания"]
  exclude_keywords: ["intern"]
  skip_test_vacancies: true             # Тесты → в файл для ручного прохождения

apply:
  max_applications: 150
  use_cover_letter: true
  cover_letter: |
    Здравствуйте!
    Меня заинтересовала ваша вакансия. Буду рад обсудить детали.
  delay_min: 1.5
  delay_max: 4.0

browser:
  headless: false
  data_dir: "~/.hh-apply"
  # proxy: "socks5://localhost:1080"    # или подхватит из env
```

## Где хранятся данные

`~/.hh-apply/`:

| Файл | Описание |
|------|----------|
| `storage_state.json` | Сессия браузера (куки) |
| `applications.db` | История откликов + пропущенные (SQLite) |
| `api_token.json` | OAuth токены для API-команд |
| `test_vacancies.txt` | Вакансии с тестами для ручного прохождения |
